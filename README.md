# Collaborative-Assignment
Uni Assignment


APPENDIX A :
ASSIGNMENT/TASK DESCRIPTION

PROJECT THEME

“Visit Malaysia 2026 (VM2026) underscores Malaysia’s commitment to sustainable
tourism development, in line with the United Nations Sustainable Development Goals
(UNSDGs). The campaign aims to showcase the nation’s diverse natural attractions,
rich cultural heritage, and vibrant tourism offerings, while driving inclusive growth
and long-term value for local communities and industry stakeholders.”
(Ref: VM2026 OFFICIALLY BEGINS: MALAYSIA ROLLS OUT A WARM WELCOME NATIONWIDE. (n.d.).
https://www.tourism.gov.my/media/view/vm2026-officially-begins-malaysia-rolls-out-a-warm-welcome-nationwide)

A. PROJECT CONSTRAINTS
• The proposed idea/project must be associated with ONE (1) of the following United Nations
Sustainable Development Goals:
o Goal 8: Decent Work and Economic Growth
o Goal 11: Sustainable Cities and Communities
o Goal 12: Responsible Consumption and Productions
• Marketplace or similar type of projects ARE NOT ALLOWED.
• The project can either be a mobile application, a web-based system, a progressive web app
(PWA), or a combination of mobile and web.

B. GENERAL INSTRUCTIONS
• This assessment requires students to work in a team. Team formation (number of team members
and member selection) will be determined by the lecturer/tutor.
• Each team must develop a software application, starting with project ideation and ending with a
demonstration of a complete and error-free prototype.
• Each team member will be assigned specific roles and responsibilities, including Project Manager,
Requirement Lead, Design Lead, Programmers and Tester. However, everybody must contribute
to each phase of the project.
• The proposed project must FOLLOW THE THEME given by the lecturer/tutor. Any project that
does not follow the theme or is considered unsuitable will be rejected.

C. PROJECT SCOPE
• The project must COMPLY with the THEME given by the lecturer/tutor.
• Each team must determine the scope, scale, and functionalities of the project:
o The project can be completed within the timeframe and duration of the project.
o The lecturer/tutor will propose additional functionalities if the project is too small.
• The project must produce a complete application/system covering all possible requirements,
logic rules, and process flows.

D. TECHNOLOGY STACK
• Each team can use any framework, library, web services, AI libraries, or programming language in
implementing the project.
• The project is not limited to any design structure or specific architecture, BUT it must have at
least THREE (3) distinctive logical layers/sections:
o The GUI – to receive input and display output to the end user
o The logical/business processing layers – to process any logic, calculation, or instruction
based on end-user input or request.
o The data processing – to handle data storage
• Depending on the project, data can be stored in any storage format (text file, device memory,
etc) deemed suitable. Usage of DBMS (eg, MySQL, Firebase, etc.) is not compulsory.
